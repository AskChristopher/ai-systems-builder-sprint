# Day 001 Reflection

## Built-in Function
`print()`

## What I Learned
Today I practiced using Python's `print()` function in several different ways. I learned how to print strings, numbers, multiple values in a single statement, customize the separator with `sep=`, change the ending with `end=`, and use f-strings to combine variables with text.

I also learned how a standard Python program is structured with a `main()` function and the `if __name__ == "__main__":` statement. While I don't fully understand why it's written that way yet, I understand that it's the standard way to organize a Python program.

## What Surprised Me
At first I thought practicing `print()` would be very easy, but printing different types of values and using the optional arguments made it more interesting than I expected.

I also noticed that GitHub Copilot completed almost all of my code after I wrote descriptive comments first. Because I understood what each practice was supposed to do, I was able to recognize that the generated code was correct instead of accepting it blindly. That showed me the value of planning before coding.

## Problems I Encountered
I wasn't sure how to run my program because the project was inside my GitHub repository. I learned how to navigate to the correct folder in the VS Code terminal and run the program with:

```powershell

```

I also learned the difference between my project's root folder and the folder containing the Python file.

## Biggest Insight
Writing comments before writing code made AI much more effective. Instead of asking AI to solve the problem for me, I defined the steps first and then evaluated the generated code. That feels much closer to building than simply copying code.
