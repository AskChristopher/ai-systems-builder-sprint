## The Augmented Learning Cycle

Observe  
↓  
Understand  
↓  
Imitate  
↓  
Modify  
↓  
Predict  
↓  
Build  
↓  
Reflect  
↓  
Teach

Learning is not complete until every stage has been attempted.

### Observe

- Read documentation.
- Study examples.
- Inspect working code.
- Understand the problem before attempting the solution.

### Understand

Explain the concept in plain language:

- What problem does this solve?
- Why does it exist?
- Where is it used?
- When should it be used?

### Imitate

- Type examples yourself.
- Never copy and paste.
- Build syntax memory through repetition.

### Modify

- Change the example.
- Experiment.
- Break it and fix it.
- Observe behavior changes.

### Predict

Before running code, predict:

- What will happen?
- What output should appear?
- Why?

Then compare your prediction with reality.

### Build

- Apply concepts to real projects.
- Create useful artifacts in every lesson.

### Reflect

Document:

- What did I learn?
- What surprised me?
- What mistakes did I make?
- What remains confusing?
- What should I practice next?

Reflection turns experience into knowledge.

### Teach

- Explain concepts as if teaching another beginner.
- If explanation is unclear, continue practicing.

Teaching is evidence of understanding.

## Study Workflow

For every topic:

1. Read `README.md` and `notes.md`
2. Run and type `examples.py`
3. Practice in `practice.py`
4. Complete prompts in `exercises.md`
5. Validate and compare with `solutions.py`
6. Record progress in `progress.md`
7. Add terms to `glossary.md` and shortcuts to `cheat-sheet.md`
