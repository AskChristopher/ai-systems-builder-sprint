# Day 003

**Date:** 2026-07-17

## What I Learned

- An exception means Python encountered something unexpected.
- Core priority is mastering `try` / `except` so programs can recover instead of crashing.
- Reading tracebacks is a professional skill: identify the exception type, the line that failed, and the message.
- Common exceptions to memorize: `ValueError`, `TypeError`, `NameError`, `IndexError`, `KeyError`, `AttributeError`, `FileNotFoundError`, `ZeroDivisionError`, `ImportError`, and `ModuleNotFoundError`.
- Advanced topics like exception groups are not urgent for current goals.

## What I Built

- Practice snippets that intentionally triggered common exceptions.
- Basic recovery patterns using `try` / `except`.
- Error-reporting examples using `except ValueError as error` and printing the exception message.

## Problems I Encountered

- Tracebacks looked overwhelming at first.
- Needed practice distinguishing between similar errors (for example, `ImportError` vs `ModuleNotFoundError`).

## Biggest Insight

- I do not need to master every advanced exception topic now; mastering core error handling and immediate practice gives the most leverage.

## Tomorrow's Focus

- Keep practicing traceback reading until I can diagnose errors quickly.
