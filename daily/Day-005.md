## What I Learned

- In Python boolean range checks, I need to repeat the variable on both sides of `and` (example: `wind >= 8 and wind <= 31`).
- `elif` always requires a condition, while `else` is the catch-all for everything not matched earlier.
- A calculator can be built with `if/elif` branches by checking the operator and mapping `+`, `-`, `*`, and `/` to the matching arithmetic.
- Staying at the current difficulty level (if/else and logical operators) is helping me build stronger fluency before moving on.
- In AI policy training design, tool availability is not the same as universal authorization; employees must verify tool, account, data type, and intended use against official company guidance.
- Strong policy training scenarios should reinforce: authorized environment, data protection, citation verification, and human accountability based on risk.
- Modern coding agents work best in a professional loop: Read -> Plan -> Implement -> Test, instead of jumping directly to code generation.
- `/init` and `CLAUDE.md` represent onboarding and durable project memory; they improve consistency across sessions by preserving architecture, conventions, and build/test commands.
- My role shifts from pure coder to engineering director: defining requirements, reviewing plans, setting quality criteria, and approving outcomes.

## Problems I Encountered

- I wrote an invalid chained condition (`wind >= 8 and <= 31`) and needed to make the second comparison explicit.
- I used `elif` in a place where `else` was the correct final fallback branch.
- Needed to tighten policy language so the game does not imply any AI tool is automatically approved for every scenario.
- Recognized a recurring risk in AI usage: asking for implementation too early can reduce quality versus requiring explicit planning first.

## Biggest Insight

- Most bugs at this stage are small syntax and control-flow mistakes, and fixing those details is what turns ideas into working programs.
- The same principle applies to policy design: precise wording changes behavior, and "AI assists, humans decide" is the core governance mindset.
- The highest leverage skill is prompt architecture for engineering workflows: better context and constraints produce better code than faster typing.
