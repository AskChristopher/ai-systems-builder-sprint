# Day 003 Reflection

## What did I learn?

- I learned that exceptions are Python's way of reporting unexpected problems.
- I practiced common exceptions: ZeroDivisionError, NameError, ValueError, and IndexError.
- I learned how try/except keeps a program from crashing and lets me show helpful messages.
- I practiced capturing the error object with except ValueError as error and printing it.

## What surprised me?

- Small one-line mistakes trigger very specific exception types.
- The same code can either crash or recover depending on whether I use try/except.

## What mistakes did I make?

- I left some practice code commented out, so parts were not actually executed at first.
- I wrote Keyerror in notes instead of the correct exception name KeyError.

## What remains confusing?

- I still want more confidence reading full traceback output quickly.
- I want more practice deciding when to catch a specific exception versus a broader one.

## What should I practice next?

- Run each exception example one by one and explain the traceback in plain English.
- Build small input exercises that use try/except for validation.
- Continue memorizing common exception names and what causes each one.
